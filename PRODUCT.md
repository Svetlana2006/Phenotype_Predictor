# Product Definition: Forensic Phenotype Predictor

## 1. Who are the users?
*   **Forensic Investigators & Law Enforcement:** Detectives handling cold cases or unidentified remains where traditional DNA profiling yields no database hits.
*   **Bioinformatics Researchers:** Scientists testing new machine learning models for phenotypic trait inference across diverse global populations.
*   **Genetics Students:** Academic users learning about the biological pathways connecting SNPs (Single Nucleotide Polymorphisms) to physical traits.

## 2. What problem does it solve?
When a forensic DNA sample is recovered from a crime scene but does not match any known profile in national databases (like CODIS), investigations hit a wall. 
Our software solves this by generating a **physical description of the unknown suspect** (Eye Color, Hair Color, Skin Color, Age, and Global Ancestry) directly from the DNA, generating actionable investigative leads.

## 3. Why is it better than existing workflows?
*   **Degradation Resilient:** Real-world forensic DNA is often highly degraded. Instead of failing when data is missing, our system dynamically hot-swaps to a **Sparse Ancestry Model** (requiring only 41 SNPs) to guarantee a result.
*   **Transparent Imputation:** Existing tools quietly guess missing genes using global averages, causing severe biases (e.g., hallucinating dark skin/eyes). Our platform actively measures **Evidence Quality** and mathematically blocks this hallucination.
*   **Explainability:** Pure AI is a "black box," which is unacceptable in court. Our engine extracts the mathematical weights of the AI and visualizes exactly *which specific genes* drove the prediction.
*   **Reproducibility:** Every prediction is accompanied by reproducibility metadata, including model version, training dataset, software version, and commit hash.

## 4. Key User Stories
*   *As an investigator*, I want to paste raw, unaligned DNA strings straight from NCBI into the system without needing to format it into complex CSV/VCF files.
*   *As a researcher*, I want to view a timeline of predictions on the same DNA sample to see if Version 2 of the AI is mathematically superior to Version 1.
*   *As a detective*, I want to instantly know the "Evidence Quality" of my sample before trusting the AI's result.
*   *As an expert witness*, I need a reproducible, stamped PDF report that lists exactly which versions of the software and models were used to generate the evidence.

## 5. Limitations
*   **Epigenetic Age requires arrays:** While pigmentation and ancestry use SNPs, Age Prediction currently relies on continuous CpG methylation arrays (Illumina 27k), which are rarely collected in standard forensic pipelines.
*   **Population Bias:** The current Ancestry model groups humans into 5 broad super-populations (AFR, AMR, EAS, EUR, SAS). It cannot pinpoint specific nationalities (e.g., "French" vs "German").
*   **Indel Complexity:** Insertions and deletions (indels) can sometimes disrupt the raw string parser if the flanking sequences mutate significantly.

## 6. Feature Roadmap (v1.1 to v2.0)
### Version 1.1 (The Professional Upgrade)
*   **Native `.vcf` Parsing:** Direct support for uploading Variant Call Format files generated by Next-Generation Sequencing (NGS) machines.
*   **Formal PDF Reports:** Automated generation of `reportlab` dossiers summarizing the DNA findings for legal submission.

### Version 2.0 (The Platform)
*   **Case Management System:** Hierarchical PostgreSQL database allowing investigators to organize predictions into "Cases".
*   **Sex & Freckle Prediction:** Training new models to include Biological Sex and Freckling density.
*   **Advanced Model Comparison:** Side-by-side ROC curves and Confusion Matrices comparing the performance of updated models.
